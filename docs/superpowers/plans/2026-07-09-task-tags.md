# Task Tags Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Obsidian-compatible tags on task documents (read every frontmatter form, write canonical flow style) plus tag-aware UI: chips on rows, click-to-filter, tags on add/edit, and a Dates | Tags grouping toggle.

**Architecture:** Core gains a lenient `note_tags` frontmatter parser + strict `is_valid_tag`, a `tags` slot on `TaskItem`/`render_task`, and one writer extension (`set_fields` consumes a block-style list when rewriting/removing its key). The shell widens `TaskDto`/`add_task`/`update_task` with validated tags. `Tasks.vue` renders chips, a single active tag filter ANDed with the title filter, tags inputs on add/editor, and a tag-section grouping mode where a task repeats under each of its tags.

**Tech Stack:** Rust (`vault_buddy_core` + Tauri shell), Vue 3 + Pinia + Tailwind 4, Vitest (`mockIPC`), `cargo test`.

## Global Constraints

- **Tag charset (write-strict, read-lenient):** letters (unicode), digits, `-`, `_`, `/`; at least one non-digit character. Reads DROP invalid entries silently; writes ERROR naming the offending token.
- **Read every Obsidian form:** flow `tags: [a, b]`, block list (`tags:` then `- item` lines), legacy comma/space string, `tag:` singular alias (consulted only when `tags:` is absent). Normalize: trim, unquote, strip a leading `#`, validate, dedupe case-insensitively keeping first-seen casing.
- **Write only canonical flow style** `tags: [a, b]`, one line, only when non-empty; clearing removes the line (or the whole block). Tags in our charset never need YAML quoting.
- **Surgical writes stay byte-preserving** except the one new rule: rewriting/removing a key whose line has an EMPTY value consumes the following `- item` block lines.
- **Rust↔TS contract:** `TaskItem`/`TaskDto` gain `tags` (`Vec<String>` / `string[]`, camelCase as-is); `add_task {id, title, due?, priority?, tags?}`; `update_task` patch gains `tags?: string[]` — absent = untouched, `[]` = clear, non-empty = replace.
- **Sort is unchanged** — tags never affect ordering. The tag-section view partitions the already-sorted, already-filtered list.
- **The shell crate (`src-tauri/src/*.rs`) does not compile on Linux.** `cd src-tauri && cargo fmt --check` is the local gate; CI compiles it.
- **Commits:** Conventional Commits; git identity `Claude <noreply@anthropic.com>` is already configured in the repo.
- **TDD:** failing test first; regression tests name their failure mode in a comment.
- Spec: `docs/superpowers/specs/2026-07-09-task-tags-design.md`.

```rust
/// True iff `s` is a valid Obsidian tag: letters (any script), digits, `-`,
/// `_`, `/`, and at least one non-digit character. Shared by the lenient
/// read-side normalization (invalid entries are dropped) and the shell's
/// strict write validation (invalid entries are an error) so the two sides
/// can never disagree on what a tag is.
pub fn is_valid_tag(s: &str) -> bool {
    !s.is_empty()
        && s.chars()
            .all(|c| c.is_alphanumeric() || matches!(c, '-' | '_' | '/'))
        && s.chars().any(|c| !c.is_ascii_digit())
}

/// Normalize one raw tag token from frontmatter: unquote, trim, strip a
/// leading `#`; None when the result fails `is_valid_tag` (dropped by the
/// lenient reader).
fn normalize_tag(raw: &str) -> Option<String> {
    let unquoted = crate::capture_note::unquote_yaml(raw.trim());
    let t = unquoted.trim();
    let t = t.strip_prefix('#').unwrap_or(t);
    is_valid_tag(t).then(|| t.to_string())
}

/// Case-insensitive dedupe preserving first-seen casing (Obsidian matches
/// tags case-insensitively but displays the authored case).
fn dedupe_tags(items: impl IntoIterator<Item = String>) -> Vec<String> {
    let mut seen = HashSet::new();
    let mut out = Vec::new();
    for t in items {
        if seen.insert(t.to_lowercase()) {
            out.push(t);
        }
    }
    out
}

/// Parse one frontmatter tags-ish key. None when the key is absent; Some of
/// the normalized (possibly empty) list when present — so a present-but-empty
/// `tags:` still shadows the `tag:` alias.
fn parse_tags_key(content: &str, key: &str) -> Option<Vec<String>> {
    let mut lines = content.lines().peekable();
    if lines.next()?.trim_end() != "---" {
        return None;
    }
    let prefix = format!("{key}:");
    while let Some(line) = lines.next() {
        if line.trim_end() == "---" {
            return None; // end of frontmatter — the body is never scanned
        }
        // Top-level keys only: an indented list item can't match (leading
        // space), same convention as note_field.
        let Some(rest) = line.strip_prefix(&prefix) else {
            continue;
        };
        let rest = rest.trim();
        let raw_items: Vec<&str> = if rest.is_empty() {
            // Block style: consume the following `- item` lines.
            let mut items = Vec::new();
            while let Some(next) = lines.peek() {
                if next.trim_end() == "---" {
                    break;
                }
                let Some(item) = next.trim_start().strip_prefix("- ") else {
                    break;
                };
                items.push(item);
                lines.next();
            }
            items
        } else {
            // Flow `[a, b]` or legacy `a, b` / `a b` string: strip optional
            // brackets, split on commas AND whitespace. A quoted item
            // containing a space would split apart — but a space is not in
            // the tag charset, so both halves are dropped by validation
            // anyway (lenient read).
            let inner = rest
                .strip_prefix('[')
                .and_then(|r| r.strip_suffix(']'))
                .unwrap_or(rest);
            inner
                .split(',')
                .flat_map(str::split_whitespace)
                .collect()
        };
        return Some(dedupe_tags(raw_items.into_iter().filter_map(normalize_tag)));
    }
    None
}

/// A task's tags from frontmatter, in every form Obsidian accepts (see
/// parse_tags_key). `tags:` wins; the `tag:` singular alias is read only
/// when `tags:` is absent. Body `#hashtags` are deliberately out of scope —
/// the scanner stays frontmatter-only like the rest of the vault domain.
pub fn note_tags(content: &str) -> Vec<String> {
    parse_tags_key(content, "tags")
        .or_else(|| parse_tags_key(content, "tag"))
        .unwrap_or_default()
}
```

```rust
/// Validate tags for a write: trim, strip a leading `#`, drop empties,
/// dedupe case-insensitively (first casing wins). Write validation is
/// STRICT where the read side is lenient — an invalid tag is an inline
/// error naming the token, so bad input can't silently vanish on save.
fn validated_tags(tags: Vec<String>) -> Result<Vec<String>, String> {
    let mut seen = std::collections::HashSet::new();
    let mut out = Vec::new();
    for raw in tags {
        let t = raw.trim();
        let t = t.strip_prefix('#').unwrap_or(t);
        if t.is_empty() {
            continue;
        }
        if !tasks::is_valid_tag(t) {
            return Err(format!(
                "Invalid tag (letters, digits, -, _ and / only; not all digits): {raw}"
            ));
        }
        if seen.insert(t.to_lowercase()) {
            out.push(t.to_string());
        }
    }
    Ok(out)
}
```
